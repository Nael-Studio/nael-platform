# Health

`@nl-framework/health` adds Kubernetes-style **liveness** and **readiness**
probes with pluggable indicators — a Terminus-equivalent. The probe routes mount
on the existing HTTP router.

## Registration

- **Liveness** (`/healthz`) returns `200` while the process is up and flips to
  `503` once shutdown begins, so rolling deploys drain correctly.
- **Readiness** (`/readyz`) runs every indicator in parallel with a per-indicator
  timeout and returns `200` only when all are up, `503` otherwise.

## Built-in indicators

| Indicator | Checks |
| --- | --- |
| `mongoIndicator()` | Pings the ORM connection (`{ ping: 1 }`) |
| `redisIndicator({ token })` | `PING`s a Redis client or cache store |
| `daprIndicator({ appId })` | `GET`s the sidecar `/v1.0/healthz` |
| `memoryIndicator({ maxRssBytes })` | Process RSS ceiling |
| `diskIndicator({ path, minFreeBytes })` | Free space on a filesystem |

Indicators run in parallel with an individual timeout — a hung dependency
reports `down` with `reason: "timeout"` rather than blocking the endpoint.
Built-ins degrade cleanly when their dependency isn't configured;
`mongoIndicator()` fails fast at **boot** when the ORM or a connection is missing.

## Security

Probe routes are mounted on an internal controller marked `@Public()`, so a
global `AuthGuard` exempts them. Keep `/healthz` and `/readyz` off public ingress
in production.
